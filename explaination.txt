The E2E flow starts with repository ingestion. Existing Robot, Python, and Playwright automation files are loaded as LangChain Documents, split into chunks, converted into transformer-based embeddings, and stored in ChromaDB. When a user gives a natural language prompt like “Create venue on Ruckus Cloud,” the FastAPI /generate endpoint triggers a LangGraph workflow. The planner creates a task plan, the retriever fetches similar historical code from ChromaDB, the generator builds a framework-specific prompt and sends it to a local LLM through Ollama, and the validator checks the generated code. Robot scripts are validated using robot dry-run, Python scripts with AST parsing, and Playwright scripts with structure checks. If validation fails, LangGraph routes the output to a fixer node, which asks the LLM to correct the code and validates again before returning the final automation script.
